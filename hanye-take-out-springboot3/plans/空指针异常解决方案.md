# 空指针异常解决方案

## 问题分析

### 错误信息
- **错误类型**: 空指针异常 (NullPointerException)
- **错误位置**: `fun.cyhgraph.controller.user.ShopController.getStatus(ShopController.java:24)`
- **错误原因**: `Cannot invoke "java.lang.Integer.intValue()" because "status" is null`

### 问题根源
在 [`ShopController.java`](server/src/main/java/fun/cyhgraph/controller/user/ShopController.java:23) 第23行，当Redis中没有"SHOP_STATUS"键时，`redisTemplate.opsForValue().get(KEY)`返回null，然后在第24行试图将null值转换为int类型，导致自动拆箱失败。

## 解决方案

### 修复代码
需要在 [`getStatus()`](server/src/main/java/fun/cyhgraph/controller/user/ShopController.java:22) 方法中添加判空处理：

```java
@GetMapping("/status")
public Result<Integer> getStatus(){
    Integer status =  (Integer)redisTemplate.opsForValue().get(KEY);
    // 判空处理：如果Redis中没有数据，默认返回0（打烊状态）
    if (status == null) {
        status = 0;
    }
    log.info("当前店铺状态为：{}", status == 1 ? "营业中" : "打烊中");
    return Result.success(status);
}
```

### 修复说明
1. **判空逻辑**: 检查从Redis获取的status是否为null
2. **默认值**: 如果为null，设置默认值为0（打烊状态）
3. **业务逻辑**: 确保后续的日志记录和返回值都不会出现空指针异常

### 预防措施
1. **初始化Redis数据**: 在系统启动时，应该初始化店铺状态
2. **异常处理**: 考虑添加全局异常处理器来处理类似的Redis操作异常
3. **数据验证**: 在关键业务逻辑中，始终进行null值检查
